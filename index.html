Stock Platform System Architecture Flow

Interactive Overview of the Automated Loop and User Interactions

I. Automated Stock Price Update Loop

AWS EventBridge

Trigger Mechanism

  • Scheduled rule triggers every **1 minute**.
  • Targets the Lambda function.
Invokes

AWS Lambda (JAR)

Data Generation & Processing

  • Runs embedded Java JAR.
  • Executes a loop 10 times for 5 seconds (1 min total).
  • Calculates new stock price values.
Updates Prices

DynamoDB (Stock Table)

Data Storage

  • Stores continuously updated stock values.
  • Accessed by Lambda (Write) and Spring Boot (Read).

II. User & API Interaction Flow

Legend: Frontend (EC2) Backend (EBS/Spring Boot) Security (JWT) RDS/DynamoDB

A. Login / Register & JWT Generation

Frontend (Login.js/AuthContext.js)

User attempts Login/Register via API call, storing the returned JWT.

Auth Request

Spring Boot (Auth API)

Validates credentials against **RDS**. Generates JWT with **Role** (User/Admin).

R/W User Details

RDS (User Details Table)

B. General Access APIs (/stocks)

Frontend (Dashboard.js)

Calls `/stocks` with **JWT**.

Spring Boot Controller

**Validates JWT.** Reads data.

DynamoDB (Stock Table)

Fetches latest stock prices.

C. User-Specific APIs

  • **Endpoints:** `/portfolio`, `/buy`, `/sell`.
  • **Frontend:** (Portfolio.js/BuySell.js) sends **JWT**.
  • **Backend Check:** Spring Boot **authorizes** only if role = **'User'**.
  • **DB Interaction:** Reads/Writes to **DynamoDB (Order/Portfolio tables)**.
Role Check

D. Admin-Specific APIs

  • **Endpoints:** `/admin/add-stocks`, `/admin/delete-users`, etc.
  • **Frontend:** (Admin Panel) sends **JWT**.
  • **Backend Check:** Spring Boot **authorizes** only if role = **'Admin'**.
  • **DB Interaction:** Performs modification/deletion across **DynamoDB** (Stocks) and **RDS** (Users).

Deployment Summary

Component Deployment Location Primary Function
React Frontend AWS EC2 Presentation, API calling, UI rendering (app.js, login.js, etc.).
Spring Boot Backend AWS Elastic Beanstalk (EBS) RESTful APIs, JWT Auth/Role-based access, business logic, ORM/Repository access.
Stock Update JAR AWS Lambda Scheduled execution for price calculation and direct DynamoDB writes.
User Details (RDS) AWS RDS Stores User/Admin data, roles, and authentication details.
Stock/Order Data (DynamoDB) AWS DynamoDB Stores high-velocity stock, order, and user portfolio transaction data.
